# Архитектура проекта Room Radio

## Общее описание

Room Radio - веб-приложение для синхронизированного прослушивания музыки в комнатах. Пользователи могут создавать комнаты, загружать аудио файлы и слушать музыку синхронно через WebSocket соединения.

## Backend компоненты

### main.py
Главный файл FastAPI приложения. Настраивает CORS, подключает роутеры, монтирует статические файлы и обрабатывает WebSocket соединения.

**Основные функции:**
- Настройка CORS для фронтенда
- Подключение REST API роутеров
- Монтирование статических файлов из `uploads/`
- WebSocket endpoint `/ws/{room_id}` для синхронизации

### models/room.py
Модели данных для комнат и треков.

**Классы:**
- `Track` - модель трека (id, filename, url, uploaded_at)
- `Room` - модель комнаты (id, name, created_at, current_track, track_position, is_playing, tracks)
- `rooms` - словарь для хранения комнат в памяти

### api/rooms.py
REST API для управления комнатами.

**Эндпоинты:**
- `POST /api/rooms` - создание новой комнаты
- `GET /api/rooms` - получение списка всех активных комнат
- `GET /api/rooms/{room_id}` - получение информации о конкретной комнате

### api/upload.py
REST API для загрузки аудио файлов.

**Эндпоинты:**
- `POST /api/rooms/{room_id}/upload` - загрузка аудио файла в комнату
- `GET /api/rooms/{room_id}/tracks` - получение списка треков в комнате

**Функции:**
- Валидация форматов файлов (mp3, wav, ogg, m4a, aac, flac)
- Проверка размера файла (максимум 50MB)
- Санитизация имен файлов
- Сохранение файлов в `uploads/{room_id}/`

### websocket/room_manager.py
Менеджер WebSocket соединений для синхронизации воспроизведения.

**Класс RoomManager:**
- `connect()` - подключение клиента к комнате
- `disconnect()` - отключение клиента
- `send_state_update()` - отправка текущего состояния новому клиенту
- `broadcast_to_room()` - отправка сообщения всем участникам комнаты
- `handle_message()` - обработка сообщений от клиентов

**Обрабатываемые события:**
- `play` - начало воспроизведения
- `pause` - приостановка
- `seek` - переход к позиции
- `track_change` - смена трека
- `position_update` - обновление позиции

## Frontend компоненты

### App.tsx
Главный компонент приложения. Управляет состоянием выбранной комнаты и отображает либо список комнат, либо плеер.

**Состояние:**
- `selectedRoomId` - ID выбранной комнаты

**Функции:**
- `handleRoomCreated()` - обработка создания новой комнаты
- `handleRoomSelect()` - обработка выбора комнаты
- `handleLeaveRoom()` - выход из комнаты

### components/CreateRoom.tsx
Компонент для создания новой комнаты.

**Функции:**
- Ввод названия комнаты
- Валидация названия
- Создание комнаты через API
- Обновление состояния в Zustand сторе

### components/RoomList.tsx
Компонент отображения списка активных комнат.

**Функции:**
- Загрузка списка комнат через API
- Автообновление списка каждые 5 секунд
- Отображение информации о комнате (название, количество треков, текущий трек)
- Переход в комнату по клику

### components/RoomPlayer.tsx
Главный компонент аудио плеера с синхронизацией.

**Функции:**
- Отображение текущего трека
- Управление воспроизведением (play/pause)
- Перемотка трека
- Управление громкостью
- Переключение между треками
- Список всех треков в комнате
- Интеграция с FileUpload для загрузки новых треков

**Использует:**
- `useRoomSocket` для синхронизации через WebSocket
- `useRoomStore` для управления состоянием
- HTML5 Audio API для воспроизведения

### components/FileUpload.tsx
Компонент для загрузки аудио файлов.

**Функции:**
- Drag & drop загрузка файлов
- Выбор файла через диалог
- Валидация формата и размера файла
- Отображение прогресса загрузки
- Обработка ошибок

### hooks/useRoomSocket.ts
Хук для работы с WebSocket соединением.

**Функции:**
- Подключение к WebSocket серверу
- Автоматическое переподключение при разрыве соединения
- Отправка событий (play, pause, seek, track_change)
- Получение и применение событий от других пользователей
- Синхронизация состояния плеера

**Возвращает:**
- `sendPlay()` - отправка события play
- `sendPause()` - отправка события pause
- `sendSeek()` - отправка события seek
- `sendTrackChange()` - отправка события смены трека

### stores/roomStore.ts
Zustand стор для управления состоянием комнаты.

**Состояние:**
- `currentRoom` - текущая комната
- `tracks` - список треков в комнате
- `currentTrack` - текущий воспроизводимый трек
- `isPlaying` - статус воспроизведения
- `position` - текущая позиция в треке

**Действия:**
- `setCurrentRoom()` - установка текущей комнаты
- `setTracks()` - установка списка треков
- `setCurrentTrack()` - установка текущего трека
- `setIsPlaying()` - установка статуса воспроизведения
- `setPosition()` - установка позиции
- `addTrack()` - добавление трека
- `reset()` - сброс состояния

### services/api.ts
API клиент для взаимодействия с backend.

**Функции:**
- `createRoom()` - создание комнаты
- `listRooms()` - получение списка комнат
- `getRoom()` - получение информации о комнате
- `uploadTrack()` - загрузка трека
- `listTracks()` - получение списка треков в комнате

## Поток данных

### Создание комнаты
1. Пользователь вводит название и нажимает "Создать"
2. Frontend отправляет POST запрос на `/api/rooms`
3. Backend создает новую комнату и возвращает её данные
4. Frontend обновляет Zustand стор и переходит в комнату

### Загрузка трека
1. Пользователь выбирает или перетаскивает файл
2. Frontend валидирует файл (формат, размер)
3. Frontend отправляет POST запрос на `/api/rooms/{room_id}/upload`
4. Backend сохраняет файл в `uploads/{room_id}/` и добавляет трек в комнату
5. Frontend обновляет список треков

### Синхронизация воспроизведения
1. Пользователь выполняет действие (play/pause/seek)
2. Frontend отправляет событие через WebSocket
3. Backend получает событие и broadcast его всем участникам комнаты
4. Все клиенты получают событие и применяют его к своему плееру

## Технологии

### Backend
- **FastAPI** - веб-фреймворк
- **WebSockets** - для синхронизации в реальном времени
- **python-multipart** - для загрузки файлов
- **Uvicorn** - ASGI сервер

### Frontend
- **React 19** - UI библиотека
- **TypeScript** - типизация
- **Vite** - сборщик и dev-сервер
- **Tailwind CSS** - стилизация
- **Zustand** - управление состоянием
- **HTML5 Audio API** - воспроизведение аудио
- **Lucide React** - иконки

## Безопасность

- Валидация форматов файлов на backend
- Ограничение размера файлов (50MB)
- Санитизация имен файлов
- CORS настройки для фронтенда
- Хранение файлов в изолированных директориях по комнатам

## Хранение данных

- Комнаты хранятся в памяти (словарь `rooms` в `models/room.py`)
- Аудио файлы хранятся локально в `uploads/{room_id}/`
- При перезапуске сервера комнаты теряются, но файлы остаются
